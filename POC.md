# POC

이 문서는 구현할 제품의 컨셉을 설명합니다.

## 요약

- 주제: 대형폐기물 품목 검색
- 설명: 입력어와 가장 유사한 대형폐기물 품목을 제안합니다.

## 맥락

대형폐기물 품목명은 고정 단어로 지정되어 있습니다. 하나의 품목을 지칭하는 단어는 여러개로 표현될 수 있으나, 지정된 단어로 검색하지 않는 경우 의미적으로 유사한 단어임에도 해당하는 품목이 검색되지 않는 문제가 발생합니다. 이를 해결하기 위해 이 제품은 입력어와 의미적으로 가장 유사한 대형폐기물 품목을 제안합니다.

예시) 입력어 `티비`와 가장 유사한 대형폐기물 품목은 `텔레비전(TV)` 입니다.

- 입력 길이는 주로 1-2 단어
- 언어는 한국어입니다.

## 프로세스

### 1. 전처리

대형폐기물정보(`waste_disposal_fee.csv`) 데이터에서 `대형폐기물명`과 `대형폐기물구분명` 열을 병합합니다.

- 형식) "<대형폐기물명> [SEP] <대형폐기물구분명>"
- 예시) "텔레비전(TV) [SEP] 가전제품"

### 2. 임베딩

`ko-sbert-sts` 모델을 통해 전처리 데이터를 임베딩 후 `.npy` 파일로 저장합니다.

### 3. 로딩

엔진 구동시 임베딩 모델과 저장한 임베딩 파일을 로드합니다.

### 4. 입력 처리

a. 사용자 입력을 받으면 모델은 입력어를 벡터화합니다.
b. 임베딩 파일 데이터와 코사인 유사도를 계산합니다.
c. 유사도가 가장 높은 결과 상위 5개를 중복 포함하여 표시합니다.

## 아키텍처

### 인터페이스

- `시도명`과 `시군구명`으로 필터링하고 사용자에게 검색 단어를 입력 받아 해당하는 품목의 수수료 정보를 표시하기 위한 웹 인터페이스
- 개발환경: TypeScript, React

### 엔진

- 인터페이스를 통해 받은 입력어를 벡터화하고 유사도를 계산, 유사도가 가장 높은 상위 결과 반환
- 개발환경: Python
